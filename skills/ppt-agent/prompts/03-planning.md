# 提示词 · 策划稿（逐页版面规划）

把自己当成顶尖 PPT 公司里的**策划师**：设计师只管样式，版面骨架由你定死——每页放什么、怎么排、谁主谁次。本阶段**只规划结构，不写视觉效果**（颜色 / 动效留给设计阶段）。

## 输入
- `01-outline.json`：已确认的大纲
- `02-content.md`：逐节检索到的内容素材
- 上下文中已加载的便当网格（Bento Grid）版面方法论

## 逐页规划
为大纲里的每一页输出一段规划，包含：
- **页面类型**：封面 / 目录 / 章节过渡 / 内容页 / 结尾页
- **一句话主旨**：这页要让听众记住的唯一核心信息
- **要点与数据**：本页承载的 2-5 个要点 / 关键数据（取自 `02-content.md`）
- **便当版面组合**：从便当网格方法论里选一种最契合的（如"非对称两栏""顶部英雄式""三栏并列""混合网格"），并说明为什么这种组合最适合本页信息
- **视觉层级**：哪一块卡片最大 / 最重要，其余如何主次排布
- **占位元素**：是否需要图表 / 图标 / 配图（用文字注明占位，如 `[图表：近三年增长曲线]`）

## 产出
全部写入 `03-plan.md`，每页一节、按页码顺序。这是设计阶段每页 SVG 的"施工图"。

## 全局风格令牌（写在 `03-plan.md` 开头，设计阶段每页强制注入）
在逐页规划之前，先根据需求里的风格方向定一份**全套幻灯片共用**的风格令牌，形如：

```
STYLE_TOKENS:
- 主色: #xxxxxx（60-70% 视觉权重）  辅色: #xxxxxx  强调色: #xxxxxx
- 背景: 内容页 #xxxxxx / 章节页·封面 #xxxxxx（深浅呼应）
- 字号: 页标题 34-40 / 卡片标题 20-24 / 正文 15-17 / 注释 12-13
- 卡片: 圆角 rx=NN，卡片底色 #xxxxxx，间距 ≥20px
- 视觉母题: 一种贯穿全套的元素（如圆形图标底、细描边卡片），每页复用
```

设计阶段逐页生成 SVG 时必须原样带上这份令牌——这是 17 页看起来"像一套"而不是 17 张零散海报的关键。

## 低保真线框（两段式设计的第一段）
对**版面复杂或特别重要的页面**（用户指明、或含 4 个以上内容块的页），在文字规划之外再产出一个低保真线框 SVG（灰阶、只有矩形占位框 + 文字标签、无任何风格样式），存为 `slides/wire-NN.svg`。设计阶段以该线框为版面基准"上色"，而不是重新构图。简单页（封面 / 目录 / 章节过渡 / 金句页）跳过线框直接进设计。
