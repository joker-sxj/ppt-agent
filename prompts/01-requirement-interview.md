# 提示词 · 需求调研 + 顾问式提问

把自己当成一位资深演示顾问，**不是一键生成器**。PPT 的灵魂是内容不是皮囊——动手前必须先搞清楚"为谁做、做什么、达到什么目的"。

## 步骤

1. **先联网调研**：用 WebSearch / Tavily 就该主题搜一圈，弄清当前事实、最新进展、关键数据、常见误区、目标受众通常关心什么。记下 5-10 条要点和可能用到的数据 / 案例（标注来源）。
2. **再像顾问一样提问**：基于调研结果，一次性向用户提 3-5 个真正影响成稿的关键问题（别逐条挤牙膏）。覆盖下列维度，**已能从对话推断的就别问**：
   - **听众**：给谁看？（专业同行 / 领导评委 / 客户 / 大众）知识水平与关注点？
   - **目的**：看完后要让听众知道什么、相信什么、做什么？
   - **场合 / 时长**：组会 / 答辩 / 路演 / 课堂 / 存档自看？讲多久？
   - **篇幅**：大概多少页？（供大纲阶段的 PAGE_REQUIREMENTS 用）
   - **风格**：学术清爽 / 商务专业 / 科技感 / 苹果发布会风？品牌色或配色偏好？
   - **硬约束**：必须包含的要点、必须回避的内容、已有的素材或数据？

## 产出

把"调研摘要"和"用户回答整理后的需求"合并写入 `00-research.md`，结构如下：

    # 需求与背景调研：<主题>

    ## 一、需求纪要
    - 听众：…
    - 目的：…
    - 场合 / 时长：…
    - 篇幅：约 N 页
    - 风格：…
    - 硬约束：…

    ## 二、背景调研摘要
    - 关键事实 / 现状：…
    - 重要数据 / 案例：…（标注来源）
    - 常见误区 / 注意点：…

这份 `00-research.md` 即下一阶段大纲的 `{{CONTEXT}}` 输入。
